Spring MVC

Es un Framework que provee la arquitectura Modelo-Vista-Controlador para usarse en el desarrollo de aplicaciones web.
Gran parte de la funcionalidad de Spring está basada en el patrón de Inyección de Dependencias

Componentes

DispatcherServlet

Es el front controller (Maneja y redirige las peticiones HTTP) de la aplicación.
Hereda de la clase javax.servlet.http.HttpServlet.
Envía la petición a los manejadores registrados para su procesamiento.

Las aplicaciones solo tiene un DispatcherServlet con el contexto de la URL raíz ( "/" )
Toda peticion dirigida a ese dominio será manejada.

Referencia :

https://howtodoinjava.com/spring5/webmvc/spring-dispatcherservlet-tutorial/

HandlerMapping

Es una interface que define una relación entre las peticiones y los controladores.
El Dispatcher Servlet lo utiliza para saber donde tiene que mandar cada petición

Referencia :

https://www.baeldung.com/spring-handler-mappings

Controller

Clase que recibe la petición del DispatcherServlet para procesarla según la lógica de negocio
Devolverá el nombre lógico de la vista a mostrar

ModelAndView

Contiene la información que se quiere mostrar en la vista.

ViewResolver

Obteniene el archivo físico (páginas JSP) que se usará para generar la vistas a partir del nombre lógico devuelto por el controlador.

Referencia :

http://acodigo.blogspot.com/2017/04/spring-mvc-viewresolver.html

View

Es la página JSP que contiene los elementos que serán procesados con la información del ModelAndView para su despliegue al usuario.

Funcionamiento

  1. El DispatcherServlet recibe una petición HTTP.
  2. El DispatcherServlet consulta al HandlerMapping para decidir a que Controller enviará la petición.
    El HandlerMapping devuelve el Controller adecuado.
  3. El Controller procesa la petición y envía un objeto ModelAndView con la información del modelo y el nombre de la vista.
  4. Con esta información el DispatcherServlet consulta al ViewResolver para resolver el nombre de la vista con el recurso de la vista.
    El ViewResolver returna la información de la View.
  5. El DispatcherServlet con esta información invoca a la vista (JSP) involucrada.
    View devuelve la página renderizada con la información del modelo.
    El DispatcherServlet devuelve una respuesta HTTP.
Referencia: http://simplej2eetutorials.blogspot.com/2014/05/spring-mvc-tutorial.html
